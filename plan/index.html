Приоритеты — что делаем

Этап 1 · стройка фронта

  1. Веб-кабинет орга: брони списком, дверной список, пуш в браузер крупная самая большая дыра (0%): без него кабинет живёт только в TG, а не все любят TG. Условие первого контакта.
  2. Напоминание гостю — включить отправку (накануне / в день) быстрая настройка уже есть, нет механизма отправки. Это главное обещание «меньше неявок» — должно работать до первого живого бара.
  3. Настройка события до конца: чат-визард заведения + ИИ-афиша средняя 40% → 100%: нужно для самообслуживания; пока онбордим руками — не блокер выхода.
  4. База гостей: экран «мои гости» + «позвать на следующее» средняя станет ценной после первых живых броней — не раньше.
  5. «Спасибо» + донат после события последняя вишенка; нужна живая посещаемость, чтобы было кого благодарить.

Этап 0 · добить хвосты данных

  1. Театр Сна: дожать подключение вебхука в Fienta пинг клиента труба задеплоена и ждёт; это будущий «кейс с цифрами» для этапа 3 — единственный хвост, который делает деньги.
  2. Парсер клубной сцены: билетки клубов (Depo и др.) не подтягиваются в базу средняя эти же клубы — первые кандидаты на страницы оргов; без билеток их страницы полупустые.
  3. Срез по оргам: Serviss-клубы вручную точечно автоматом не берётся (SPA виснет); проверять Pixel Helper-ом при подготовке конкретного мини-аудита, не заранее.
Поверхности этапа 2 (страница орга, мини-аудит, демо-дэшборд) собираются из П1–П3 этапа 1 и хвостов этапа 0 — отдельных задач не заводим, они «выпадают» по мере готовности компонентов. Внутренний рабочий документ; проценты обновляются по мере готовности.
